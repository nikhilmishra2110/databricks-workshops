# Rubric: Learner SQL Submission

| Criteria | Strong | Developing | Needs Help |
| --- | --- | --- | --- |
| Correctness | Query answers the question fully | Query is close but misses a condition or edge case | Query does not answer the question |
| Schema Use | Uses the right tables and keys | Uses right tables but weak join/filter choice | Uses unrelated tables or keys |
| Clarity | Query is readable and consistently formatted | Query works but is hard to scan | Query is difficult to understand |
| Explanation | Learner can explain each clause | Learner can explain the general idea | Learner cannot explain the query |
| Debugging | Learner can identify and fix the issue | Learner can fix with hints | Learner is blocked without solution |

## Feedback Pattern

Use this structure:

1. What is correct.
2. The main issue to fix.
3. One concrete next step.
4. Optional stretch improvement.
