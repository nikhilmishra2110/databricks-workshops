# Repo Operating Model

## Asset Lifecycle

1. Draft: created for a specific workshop or idea.
2. Tested: used locally or reviewed by another facilitator.
3. Run: used in a live workshop.
4. Revised: updated from feedback and facilitator notes.
5. Reusable: stable enough to copy into future workshops.

## Folder Strategy

Use `prompts/` for reusable prompt cards.

Use `workshops/` for complete workshop packages.

Use `examples/` for small, runnable starter workshops.

Use `docs/`, `checklists/`, and `rubrics/` for guidance that applies across workshops.

## Versioning Guidance

For breaking changes to a workshop:

- create a new workshop folder if the audience, database, or goals change substantially
- keep the older version if it has already been used successfully
- document major changes in the workshop README

## Privacy Standard

Treat workshop assets as shareable by default. Do not commit:

- real participant names
- client-specific schemas unless approved
- production data
- database credentials
- screenshots with private information
