# Publishing Checklist

Use this before making the repo public or sharing a workshop externally.

- [ ] Repository has a license
- [ ] README explains repo purpose and structure
- [ ] No secrets, credentials, or private data are committed
- [ ] Example datasets are synthetic
- [ ] Prompts use placeholders for sensitive inputs
- [ ] Runnable examples include setup instructions
- [ ] GitHub issue templates fit the workflow
- [ ] Pull request template includes validation checks
- [ ] Large generated files are excluded or stored outside git
